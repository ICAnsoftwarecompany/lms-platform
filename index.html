التواصل

شات داخلي لحظي يقلل الاعتماد على القنوات الخارجية ويضمن سياقًا واضحًا للمحادثات.

الوحدات الرئيسية في النظام

كل وحدة هنا مستخرجة من تفاصيل المشروع، ومصممة لتوضح نطاق العمل الكامل بشكل مهني.

لوحة مسؤول النظام

  • إدارة الحسابات وإنشاء الأدوار المختلفة.
  • إدارة المناهج من إنشاء إلى مستويات ووحدات ودروس واختبارات.
  • إدارة الكورسات والطلاب والمدرسين والشركاء والحسابات.
  • متابعة طلبات تجميد الحصص والتقييمات والمهام.

رحلة الطالب

  • كورساتي مع الحصص القادمة والمنتهية ومحتوى الدروس.
  • مهام مع رفع الحل وتعديل الملفات ومتابعة تقييم المدرس.
  • اختبارات متاحة مع مراجعة المحاولات السابقة.
  • تقييمات للحصص والمدرسين والشات عند الحاجة.

رحلة المدرس

  • لوحة تعرض الكورسات والحصص والطلاب.
  • بدء الحصة وتسجيل الحضور وإضافة الملاحظات.
  • تقييم مهام الطلاب وتعديل التقييمات.
  • متابعة المستحقات المالية والتواصل الداخلي.

رحلة ولي الأمر

  • إضافة الأبناء أو مراجعة بياناتهم الحالية.
  • تصفح المناهج والخطط التعليمية المتاحة.
  • طلب كورس جديد وتحديد اليوم والوقت والدفع.
  • مراجعة تقارير الطالب والتواصل مع الإدارة.
الرحلة التشغيلية هنا متسقة: الطالب يتعلم، المدرس يدير الجلسة، ولي الأمر يتابع، الإدارة تشرف، والدعم يتدخل عند الحاجة، وكل ذلك داخل منصة واحدة.

النظام المالي بالتفصيل

تم تصميم البنية المالية لتكون مركزية وقابلة للتتبع، مع سجل واضح لكل حركة مالية.

حسابات المدرسين

  • إجمالي الرصيد والمستحقات والمدفوعات.
  • إضافة دفعات وربطها بالبنك والتاريخ والوصف والمرفقات.
  • احتساب المستحقات حسب الساعة أو الحصة أو العقد أو النسبة.
  • ربط المستحقات بجلسات فعلية مغلقة.

حسابات الشركاء - التعاون

  • عرض الرصيد الكلي والمدفوع والمتبقي لكل شريك.
  • إضافة دفعة مع البنك والعملة والتاريخ والوصف والمرفق.
  • تقارير مالية تشمل الدائن والمدين وصافي الحساب.
  • إدارة المعاملات المالية الخاصة بالشريك من لوحة الإدارة.

البنوك والتحويلات

  • إدارة الخزن أو الحسابات البنكية وتفاصيل كل بنك.
  • تنفيذ تحويلات بين البنوك مع مصدر ووجهة العملية.
  • إمكانية عكس التحويل عند الحاجة مع حفظ الأثر.
  • ربط المعاملات بالبنوك لضمان تتبع دقيق للتدفقات.

مدفوعات الطلاب والاسترداد

  • تسجيل المعاملات المرتبطة بالطالب والكورس والبنك.
  • دعم الاسترداد مع السبب والتاريخ والمرفقات.
  • عرض معاملات الكورس والطلاب المرتبطين به.

المصروفات والتقارير العامة

  • تسجيل المصروفات التشغيلية أو الإدارية.
  • تصنيف المعاملات كإيداع أو سحب أو مصروف أو دفع لطرف محدد.
  • تقارير عامة تعرض الداخل والخارج والصافي خلال فترة محددة.

ملخص الوحدات المالية

  • مدفوعات المدرسين: المبلغ، نوع الدفع، الشهر المالي، الحالة.
  • مدفوعات الشركاء: الشريك، البنك، العملة، الوصف، الملف.
  • مدفوعات الطلاب: الطالب، الكورس، البنك، العملة، المرفق.
  • التحويلات والتقارير: الداخل، الخارج، الصافي، التفاصيل.

نظام الشات والتواصل الداخلي

جزء أساسي من المنصة لتقليل القنوات الخارجية وتوحيد التواصل بين الإدارة وجميع الأطراف.

أنواع المحادثات

  • محادثات فردية بين مستخدمين حسب الصلاحيات.
  • محادثات جماعية تضم أكثر من طرف.
  • محادثات دعم بين المستخدم وفريق الدعم.
  • محادثات إدارية بين الإدارة والمدرسين أو الشركاء أو الدعم.

مكونات الواجهة

  • Chat Sidebar لعرض القائمة والبحث والرسائل غير المقروءة.
  • Chat Window لعرض الرسائل زمنياً.
  • Chat Input للكتابة والإرسال وإرفاق الملفات.
  • Floating Chat Button للدخول السريع مع عداد الرسائل.

آلية العمل

  1. 1. تحديد المسار يتم جلب المحادثات حسب نوع المستخدم والصلاحيات.
  2. 2. تحميل الرسائل عند فتح المحادثة يتم جلب الرسائل السابقة وترتيبها زمنياً.
  3. 3. البث اللحظي الرسائل الجديدة تُحفظ ثم تُبث للطرف الآخر فوراً.
  4. 4. الاحتياط في حال تعذر الاتصال اللحظي يعمل Polling كبديل.

الصلاحيات المقترحة

  • Admin: مع جميع الأدوار وإدارة المجموعات.
  • Student: مع الدعم والمدرسين والإدارة عند الحاجة.
  • Teacher: مع الإدارة والدعم وطلاب الكورسات.
  • Parent / Partner / Support: حسب نطاق كل حالة تشغيلية.

نسخة تطبيق الموبايل

البنية المقترحة تخدم Android و iOS من نفس الـ API ونفس منطق الصلاحيات، بدل بناء نسخة منفصلة.

خصائص تطبيق الموبايل

  • تسجيل الدخول والتسجيل وتأكيد OTP من الهاتف.
  • لوحات مبسطة لكل نوع مستخدم.
  • عرض الكورسات والحصص والاختبارات والمهام للطلاب.
  • متابعة الأبناء وطلب الكورسات لأولياء الأمور.
  • إدارة الحصص وتسجيل الحضور للمدرسين من الهاتف.
  • متابعة الرصيد والتقارير للشركاء.
  • استقبال تنبيهات الرسائل والعمل داخل الشات الداخلي.

رسالة مهمة للعميل

الهدف ليس بناء واجهة ويب فقط، بل تأسيس نظام يخدم الويب والموبايل بنفس القاعدة التشغيلية، ما يقلل التكرار ويرفع الاستقرار ويجعل التوسع لاحقاً أسهل.

الهوية البصرية في هذه الصفحة تحاكي فكرة المنصة نفسها: منصة مؤسسية، منظمة، عالية الوضوح، وقابلة للتوسع من دون فقدان الانطباع الاحترافي.

الأمان والصلاحيات

يتم فصل الصلاحيات وحماية المسارات لضمان وصول كل مستخدم لما يخصه فقط.

المصادقة

  • تسجيل دخول وحفظ Token للمستخدم.
  • توجيه تلقائي بعد تسجيل الدخول حسب نوع الحساب.
  • إرسال API Password وفق آلية النظام الحالية.

حماية البيانات

  • حماية المسارات من الوصول غير المصرح به.
  • التحقق من البيانات قبل الإرسال إلى الـ API.
  • تسجيل العمليات المهمة مثل المدفوعات والتحويلات والتعديلات المالية.

فصل الصلاحيات

  • Admin و Student و Teacher و Support و Parent و Partner.
  • كل دور يرى صفحاته وبياناته فقط حسب الصلاحيات.
  • تقليل الأخطاء التشغيلية وتحسين تجربة الاستخدام.

سيناريوهات التشغيل الرئيسية

هذه السيناريوهات توضح كيف تدور العمليات الأساسية داخل النظام بشكل عملي ومترابط.

رحلة الطالب الجديد

  1. 1. إنشاء حساب بالبريد أو رقم الهاتف.
  2. 2. استقبال OTP للتحقق من الحساب.
  3. 3. تسجيل الدخول ثم الانتقال إلى الكورسات أو المناهج المتاحة.
  4. 4. اختيار المنهج أو الخطة ومتابعة الحصص والاختبارات والمهام.

رحلة ولي الأمر

  1. 1. الدخول إلى لوحة التحكم.
  2. 2. إنشاء حساب طالب لأحد الأبناء أو مراجعة الأبناء الحاليين.
  3. 3. تصفح المناهج والخطط المتاحة.
  4. 4. اختيار الطالب والخطة واليوم والوقت وطريقة الدفع ثم الإرسال.

رحلة الإدارة في إنشاء منهج وخطة

  1. 1. إنشاء منهج جديد وتحديد البيانات الأساسية.
  2. 2. إضافة المستويات والوحدات والدروس والاختبارات.
  3. 3. إضافة خطط تسويقية مرتبطة بالمنهج.
  4. 4. تحديد الاسم والنوع وطريقة الحضور والسعر التعليمي والمميزات.

رحلة مالية مختصرة

  1. 1. تسجيل معاملات الطلاب أو الشركاء أو المدرسين من لوحة الإدارة.
  2. 2. ربط كل معاملة ببنك أو خزنة وتاريخ ووصف وملف عند الحاجة.
  3. 3. التقارير تعرض الداخل والخارج والصافي خلال فترة محددة.
  4. 4. مراجعة تفاصيل كل طرف ودفعاته وحركاته المالية.

المخرجات النهائية ومعايير القبول

هذه هي النتيجة المطلوبة من التنفيذ كما وردت في المستند، بصياغة واضحة وقابلة للتقديم للعميل.

المخرجات المتوقعة

  • منصة ويب كاملة بواجهات مخصصة لكل دور.
  • تطبيق موبايل Android و iOS بنفس المنطق والصلاحيات.
  • لوحات تحكم لكل نوع مستخدم.
  • نظام إدارة المناهج والكورسات والخطط.
  • نظام مالي شامل للمدرسين والشركاء والطلاب والبنوك.
  • نظام شات داخلي لحظي ومنظم.
  • تكامل كامل مع الـ APIs والوسائط والملفات.
  • واجهة عربية RTL احترافية ومتجاوبة.

معايير القبول

  • كل دور يرى صفحاته وبياناته فقط حسب الصلاحيات.
  • تنفيذ دورة الطالب من التسجيل حتى الكورس والمهام والاختبارات.
  • تنفيذ دورة المدرس من عرض الحصص حتى تقييم المهام.
  • متابعة ولي الأمر للأبناء وطلب كورسات جديدة.
  • إدارة الحسابات المالية والتقارير من لوحة الإدارة.
  • عمل الشات بين المستخدمين وفق الصلاحيات المحددة.
  • عمل نسخة الموبايل على نفس البيانات والمنطق التشغيلي.
  • ظهور الرسائل والأخطاء والتنبيهات بشكل واضح.
